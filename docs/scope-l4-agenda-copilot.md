## 0. L4 是什么（钉死，不翻案）

研究议程副驾——**管多个方向 / 找隐藏联系 / 预警威胁，人在环内**。

- **L4 ≠ auto-research**（铁律）。auto-research 把人踢出环 → 杀轨迹护城河 + 挤烧钱赛道（Sakana/FutureHouse/Google）必输 + 违背"逼你自己想清楚"的魂。L4 是**副驾**，不是自动研究员。
- 前置：**"多文档才可管"**——L4 操作一整个研究计划（多 Project / 多 artifact），不是单个想法。

## 1. L4 第一刀（拍定）= 跨议程的隐藏联系 / 威胁预警

本质 = **对语料图的跨 Project 遍历 + 威胁度排序**。"管多方向"作为承载它的视图层后叠。

### 三类威胁（锁定）
1. **跨方向矛盾**：A 方向的 claim 撞了你 B 方向 survived 的 claim。
2. **地基被别处 kill**：某方向**依赖**的 claim 在别处被 kill → 整条线悬空。
3. **共性缺口/方法**：多条方向反复撞同一未解 gap / 同一可疑方法。

### 威胁度排序
**威胁度 = severity × centrality**（L2 张力排序的现成设计）。
- **severity** = 多致命：精准撞车（杀新颖性）、反驳核心假设（杀有效性）最高。
- **centrality** = **图上的度数**：被很多 claim `builds-on` 的节点 = 核心，它被 kill 威胁一大片。
- **过阈值才主动高亮，弱的沉面板当 FYI**。

## 2. 怎么露面（守人在环 / 取证不定见，拍定）

- **触发 = 显式"扫一遍我的议程"**（按需，像 ① 评品味），**不是每次 grill 自动跑**——议程级图分析贵、且是刻意的周期性 review。可配被动"威胁数"角标。
- **呈现 = 按威胁度排的取证清单**，每条一句事实陈述 + 受影响节点；过阈值高亮、弱的 FYI。
- **守线**：L4 只**摆威胁 + 排序**，**绝不替你重排议程 / 砍方向 / 自动行动**。你看完自己决定（处理 / 接受风险 / 砍）。取证不定见。

## 3. → ③ Tier 1 图需求（本次 scoping 的实际产出）

L4 反推出 Tier 1 持久语义图**必须存/能算**的东西：

- **边集**：
  - `contradicts`（**② 已有**，Tier 0 图里已在；Tier 1 让它持久 + 主动算，不止 grill 时）。
  - `builds-on / depends-on`（**Tier 1 新增的 LLM 语义边**）—— 支撑"地基被别处 kill"威胁 + centrality 度数。
  - `shares-method / shares-gap`（**Tier 1 新增**）—— 支撑"共性缺口/方法"威胁。
- **跨 Project 可遍历**：边要能跨方向（claim 带 project 归属；威胁本质是跨方向的）。
- **claim status 在图上**（survived/killed/open）—— 威胁判断要看节点死活。
- **可算 centrality**（节点度数）—— 威胁度排序要它。
- 仍**不碰 embedding**（Tier 0/1 铁律）；边是 LLM 标的类型化关系，结构化存。

> 一句话：**Tier 1 = 把 Tier 0 的结构图，加上 LLM 算的 `builds-on/shares-*` 语义边并持久化、跨 Project、可算度数**——正好是 L4 威胁检测的底座。建 Tier 1 时以此为靶。

## 4. 已知问题（建 L4 时必须先回答，2026-07-13 记）

- **语义边承重前需硬闸**。Tier 1 的 `builds-on/shares-*` 边是 LLM 软输出、「出生 confirmed 可撤回」——今天只承担展示，无妨；但 L4 拿 centrality（度数）做威胁度排序那一刻，未经人核的边就开始**承重**（软输出进环）。届时二选一，不拍不建：
  1. 边加确认状态分层——只有人核过的边计入度数；
  2. L4 输出显式标注——「本条威胁坐在 N 条未验证边上」，把不确定性摆到台面。
  - 注：`contradicts`/`grounds`/`narrowed_from` 是确定性来源（confirmed 事件 / verdict payload），不受此限；受限的只有四类 LLM 语义边。
